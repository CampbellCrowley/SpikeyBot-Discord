Class: Player

HungryGames~Player(id, username, avatarURL, nicknameopt, nullable)

Serializable container for data pertaining to a single user.

Constructor

new Player(id, username, avatarURL, nicknameopt, nullable)

Parameters:
Name Type Attributes Default Description
id string The id of the user this object is representing.
username string The name of the user to show in the game.
avatarURL string URL to avatar to show for the user in the game.
nickname string <optional>
<nullable>
 null The nickname for this user usually assigned by the guild. If the user does not have a nickname, this will have the same value as `name`.
Properties:
Name Type Description
id string The id of the User this Player represents.
name string The name of this Player.
avatarURL string The URL to the discord avatar of the User.
nickname string The nickname for this user usually assigned by the guild. If the user does not have a nickname, this will have the same value as `name`.
living boolean Is the player still alive.
bleeding number How many days has the player been wounded.
rank number The current rank of the player in the game.
state string The current player state (normal, wounded, dead, zombie).
kills number The number of players this player has caused to die.
weapons Object.<number> The weapons the player currently has and how many of each.
settings Object Custom settings for this user associated with the games.
dayOfDeath number The day at which the player last died in the game. Only a valid number if the player is currently dead. Otherwise a garbage value will be available.
